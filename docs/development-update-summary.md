# 개발 업데이트 요약

작성일: 2026-06-10

이 문서는 최근 앱 개선 작업과 펌웨어 `1.0.7` 업데이트 내용을 기획자 관점에서 쉽게 확인하기 위한 기록입니다.

## 이번 펌웨어 1.0.7 업데이트

### 적용 상태

- USB로 연결한 테스트 기기 1대에 `1.0.7` 펌웨어를 직접 설치했습니다.
- 서버 전체 배포는 아직 하지 않았습니다.
- 서버의 최신 펌웨어 정보는 아직 `1.0.6`입니다.
- 따라서 다른 기기나 다른 앱 사용자에게는 아직 `1.0.7`이 노출되지 않습니다.

### 주요 개선 내용

- 기기 제어 보안 강화
  - 새로 등록되는 기기는 전용 확인 키를 생성합니다.
  - 앱은 이 확인 키를 저장하고, 물 공급/팬 작동/펌웨어 업데이트 명령을 보낼 때 함께 보냅니다.
  - 기기는 확인 키가 맞는 명령만 받아들이도록 준비됐습니다.

- 펌웨어 업데이트 파일 검증 추가
  - 서버 업데이트 방식에서는 `md5`라는 파일 검증값이 필요합니다.
  - 기기는 펌웨어 파일을 받은 뒤, 받은 파일이 서버가 안내한 파일과 같은지 확인합니다.
  - 파일이 깨졌거나 다른 파일이면 업데이트를 중단합니다.

- 물 공급/팬 수동 작동 안전 제한 정리
  - 앱 명령뿐 아니라 MQTT 명령으로 물 공급이나 팬을 켜도 장시간 켜진 채 방치되지 않게 했습니다.
  - 현재 수동 작동은 60초 제한 정책을 따릅니다.

- 초기화 범위 정리
  - 기기 Wi-Fi 정보를 초기화할 때 제어용 확인 키도 같이 삭제되도록 정리했습니다.
  - 이전 사용자의 제어 정보가 남지 않도록 하기 위한 조치입니다.

### 사용자님이 체감하는 변화

- 앱에서 물 공급이나 팬을 누르는 사용 방식은 거의 같습니다.
- 테스트 기기는 새 펌웨어로 동작하지만, 장치 등록은 다시 필요했습니다.
- 보안 구조가 강화되어 앞으로 "등록된 앱이 등록된 기기를 제어한다"는 기준을 만들 수 있습니다.

## 이전 앱 개선 작업

### 홈 화면과 물 공급 표시 추가 개선

- 물 공급 카드의 배경 비디오 파일을 교체했습니다.
- 물 공급이 꺼져 있을 때는 배경 이미지/영상에만 약한 흐림 효과가 들어가고, 텍스트와 상태 배지는 또렷하게 보이도록 조정했습니다.
- 물 공급주기, 팬 작동, 물 공급 카드의 세로 크기를 조금 키워 한 화면 안에서 더 안정적으로 보이게 했습니다.
- 물 공급주기 게이지는 펌웨어의 실제 자동 분사 주기인 2시간 기준으로 다시 정리했습니다.
- 카운트다운이 끝나고 기기가 "곧 분사" 상태를 보고하기 전까지 짧은 공백이 생겨도 화면이 "예약 없음"으로 깜빡이지 않게 했습니다.
- 데모 기기는 물 공급주기 중간쯤 지난 자연스러운 상태로 보이게 조정했습니다.

### 홈 화면 개선

- 등록된 기기가 있을 때 홈 화면을 대표 기기 중심 구조로 변경했습니다.
- 제품 사진, 제품명, 위치, 연결 상태, 물 공급주기, 팬 작동, 물 공급, 성장 진행을 카드형으로 정리했습니다.
- 물 공급주기 원형 게이지를 추가했습니다.
- 팬 작동 카드에는 공기 흐름 애니메이션을 적용했습니다.
- 물 공급 카드는 추후 방향 확정을 위해 제목과 상태 중심으로 정리했습니다.

### 장치 추가 화면 개선

- 제품 선택 화면을 가로 스크롤 구조로 변경했습니다.
- 제품 이미지가 중앙에 보이도록 레이아웃을 조정했습니다.
- 다음/이전 버튼 스타일을 공통 버튼 스타일로 맞췄습니다.
- 기기 Wi-Fi 선택 목록을 앱 내부에서 보여주도록 개선했습니다.
- 기기 Wi-Fi 연결이 확인되면 별도 확인 버튼 없이 다음 단계가 활성화되도록 바꿨습니다.

### 버튼과 햅틱 개선

- 알림 버튼, 장치 추가 버튼, 팝업 CTA 버튼 스타일을 정리했습니다.
- 공통 버튼 컴포넌트를 만들어 같은 질감과 반응을 재사용할 수 있게 했습니다.
- 버튼 클릭 시 햅틱 반응을 추가했습니다.

### 상태/데이터 안정화

- 앱 시작 직후 저장된 기기 목록이 빈 목록으로 덮어써질 수 있던 문제를 수정했습니다.
- 저장되는 기기 정보에서 온라인/팬/물 공급 같은 실시간 상태값을 제외하고, 기기 이름/위치/토큰/IP처럼 오래 보관해야 하는 값만 저장하도록 정리했습니다.
- 저장된 데이터가 깨진 경우에는 원본을 백업한 뒤에만 새 저장을 허용하도록 바꿨습니다.
- 저장된 기기 목록 중 일부 항목만 깨진 경우에는 정상 기기는 살리고, 손상된 정보는 백업하도록 개선했습니다.
- 저장소를 잠깐 읽지 못한 경우에는 바로 덮어쓰지 않고, 재시도 후 기존 기기와 앱 실행 중 추가된 기기를 합치도록 처리했습니다.
- 저장소 읽기/쓰기 오류와 손상 정보 복구 상태를 홈 화면 배너로 안내하고, 사용자가 다시 시도할 수 있게 연결했습니다.
- 기기를 추가한 직후 바로 삭제했을 때 저장 순서가 꼬여 삭제한 기기가 다시 살아날 수 있던 문제를 수정했습니다.
- 데모 기기는 실제 사용자 기기가 아니므로 저장 대상에서 제외했습니다.
- 주기적으로 기기 상태를 확인할 때 오래된 목록이 최신 상태를 덮어쓰지 않도록 병합 방식을 개선했습니다.
- 5초마다 상태를 확인해도 같은 물 공급 예약 시간을 다른 시점에서 본 것이라면 불필요한 저장과 전체 화면 갱신이 일어나지 않도록 조정했습니다.
- WebSocket으로 같은 상태가 반복 수신될 때도 불필요한 화면 갱신이 일어나지 않도록 추가 정리했습니다.
- WebSocket 서버에서 잘못된 상태 메시지가 와도 앱이 멈추지 않도록 메시지 검증을 추가했습니다.
- 숫자값이 잘못 들어오는 서버 상태 메시지는 `NaN`이 앱 상태에 남지 않도록 정규화했습니다.
- 물 공급 카운트다운 기준 시간과 기기의 마지막 응답 시간을 분리했습니다. 그래서 정상적으로 기기 상태가 들어오는 중에는 연결 상태가 온라인/오프라인으로 깜빡이지 않습니다.
- WebSocket 명령을 보낼 때도 최근 기기 응답 시간을 따로 보고, 불필요한 재연결로 명령이 늦어지는 상황을 줄였습니다.
- 선택된 기기는 기기 전체 정보를 복사해서 들고 있지 않고, 선택된 기기 ID 기준으로 현재 목록에서 다시 찾도록 정리했습니다.
- WebSocket에서 미점유 온라인 기기를 자동으로 내 기기로 묶는 동작을 제거했습니다.
- 팬/물 공급 카드가 더 이상 임시 고정값이 아니라 실제 기기 상태를 기준으로 표시되도록 했습니다.
- 팬 작동 애니메이션은 팬이 꺼져 있을 때 반복 계산을 멈추도록 분리했습니다.
- 펌웨어 업데이트 표시는 서버 버전이 실제로 더 높고, `md5` 파일 검증값이 있을 때만 "업데이트 가능"으로 보이게 했습니다.

### 테스트와 개발 안정성

- `useDevices`, `deviceWebSocket`, `deviceCommands` 관련 테스트를 추가했습니다.
- 물 공급주기 표시, 펌웨어 버전 비교, 저장 안정성, WebSocket 연결 안정성 테스트를 추가했습니다.
- 현재 기준 `tsc`, `eslint`, Jest 51개 테스트가 통과합니다.
- Jest 설정을 정리해 현재 테스트가 통과하도록 만들었습니다.
- Metro 로그, 임시 스크린샷, zip 파일, 오래된 펌웨어 사본을 정리했습니다.
- `yarn.lock`은 제거하고 `package-lock.json` 기준으로 정리했습니다.
